# System Patterns

## Архитектура

Основные модули:
- `core/` - ядро генерации паттернов
- `ui/` - пользовательский интерфейс
- `presets/` - управление предустановками
- `utils/` - утилиты для работы с файлами и MIDI

Текущий UI-паттерн:
- Главная страница построена как одностраничный экран с hero-секцией, панелью управления, областью piano roll и боковой аналитикой.
- `src/index.ts` отвечает за инициализацию интерфейса, синхронизацию контролов и рендер информационной панели.
- `src/ui/gridRenderer.ts` рендерит piano roll напрямую через DOM и inline-стили, используя CSS-переменные из `style.css`.
- `style.css` использует однотонные акцентные кнопки и общую систему CSS-переменных для темной и светлой темы.
- `src/ui/gridRenderer.ts` больше не строит диапазон строк по общему диапазону нот; он рендерит только реально присутствующие в паттерне строки `pitch/octave`.

Текущий паттерн генерации:
- `src/core/generator.ts` формирует мелодию по шагам, учитывая сильные доли, середину фразы и завершающее разрешение в тонику.
- Октавы меняются ограниченно, чтобы визуальный рисунок piano roll был читаемым и не выглядел случайной россыпью.
- Для коротких паттернов используется отдельный предсказуемый контур, чтобы 4-шаговые схемы всегда были заполнены и музыкально читались как тоника -> терция -> квинта -> разрешение.

## Связи подсистем

- generator.ts зависит от harmony.ts и scale.ts
- ui/依赖于 core/ для генерации паттернов
- midiExporter.ts использует models/pattern.ts
- Визуальная тема задается глобальными CSS-переменными и автоматически переключается через `prefers-color-scheme`.
