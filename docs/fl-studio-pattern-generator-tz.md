# Техническое задание: Генератор визуальных паттернов для FL-Studio

## 1. Введение

Генератор визуальных паттернов для FL-Studio создает визуальные узоры, соответствующие гармоничным музыкальным комбинациям. Система позволяет настраивать тональность (мажор/минор), длину паттерна и BPM, генерируя визуальные элементы, синхронизированные с музыкальной структурой.

## 2. Глоссарий

- **FL-Studio**: Цифровая станция звукозаписи (DAW) от Image-Line
- **Визуальный паттерн**: Графическое представление музыкальной структуры в виде узора или ритмического дизайна
- **Тональность**: Мажорная или минорная гамма, определяющая гармонический характер
- **BPM (Beats Per Minute)**: Скорость воспроизведения, измеряемая в ударах в минуту
- **Паттерн длины**: Количество тактов или шагов в последовательности
- **Гармоничная комбинация**: Сочетание нот, образующее приятное на слух музыкальное звучание

## 3. Основные требования

### 3.1 Настройка тональности (мажор/минор)
- Поддержка двух режимов: мажор и минор
- Автоматическая генерация нот в выбранной гамме
- Возможность переключения в реальном времени

### 3.2 Настройка длины паттерна
- Диапазон: 1-64 шага
- Изменение длины без потери тональности и BPM
- Визуальная адаптация сетки под выбранную длину

### 3.3 Настройка BPM
- Диапазон: 40-240 BPM
- Изменение скорости без перегенерации паттерна
- Отображение текущего BPM в интерфейсе

### 3.4 Генерация гармоничных паттернов
- Использование только нот из выбранной тональности
- Избегание диссонирующих интервалов (минорная секунда, увеличенная четверть)
- Применение правил гармонии для созвучных комбинаций
- Автоматический выбор аккордовых нот (1, 3, 5 ступени)

### 3.5 Визуальное представление
- Сеточное представление (64 колонки × 12 строк)
- Альтернативная круговая диаграмма
- Цветовая кодировка нот
- Интерактивное редактирование (клик по ячейке)

### 3.6 Экспорт в FL-Studio
- Экспорт в формате MIDI (Type 1)
- Сохранение тональности, длины и BPM
- Поддержка стандартного MIDI файлового формата

### 3.7 Сохранение и загрузка
- Формат .fpp (FL Studio Pattern Project)
- Сохранение всех параметров: тональность, длина, BPM, ноты
- Восстановление сохраненных паттернов

### 3.8 Пресеты и шаблоны
- Минимум 5 предустановленных паттернов
- Возможность сохранения пользовательских пресетов
- Экспорт/импорт пресетов

### 3.9 Валидация
- Проверка диапазона BPM (40-240)
- Проверка длины паттерна (1-64)
- Проверка тональности (мажор/минор)
- Отображение ошибок в реальном времени

### 3.10 Производительность
- Генерация паттерна менее чем за 500ms
- Отзывчивость интерфейса при работе с паттернами до 1000 нот
- Индикатор загрузки для операций >1 секунды

## 5. Модели данных

### Pattern
```typescript
interface Pattern {
  id: string;
  notes: Note[];
  length: number;        // 1-64
  bpm: number;           // 40-240
  tone: Tone;            // 'major' | 'minor'
  createdAt: Date;
  updatedAt: Date;
}
```

### Note
```typescript
interface Note {
  pitch: number;         // 0-12 (номер в гамме)
  octave: number;        // Октава
  duration: number;      // Длительность в шагах
  velocity: number;      // 0-127 (сила нажатия)
  color: string;         // Цвет для визуализации
}
```

### GenerationOptions
```typescript
interface GenerationOptions {
  tone: Tone;
  length: number;
  bpm: number;
  complexity?: number;   // 0-100
  randomness?: number;   // 0-100
}
```

## 6. Алгоритмы генерации

### Гаммы
- **Мажорная**: [0, 2, 4, 5, 7, 9, 11]
- **Минорная**: [0, 2, 3, 5, 7, 8, 10]

### Правила гармонии
1. Избегать последовательных диссонирующих интервалов
2. Использовать ступени гаммы последовательно (ход по ступеням)
3. Чередовать длинные и короткие ноты для ритмического интереса
4. Использовать аккордовые ноты (1, 3, 5 ступени) чаще для гармонической стабильности

### Консонантные интервалы
[0, 3, 4, 5, 7, 8, 9, 12] — унисон, мин. 3-я, маж. 3-я, 4-я, 5-я, мин. 6-я, маж. 6-я, октава

### Диссонантные интервалы (запрещены)
[1, 6, 11] — мин. 2-я, ув. 4-я, мин. 7-я

## 7. Интеграция с FL-Studio

### MIDI экспорт
- Формат: MIDI Type 1 (мульти-трековый)
- Трек 1: Мелодическая информация
- Трек 2: Ритмическая информация (опционально)
- PPQ: 96
- Tempo: 60000000 / BPM (микросекунды на такт)

## 9. Технологии

- **MIDI**: midi-file или аналогичная библиотека
- **Тесты**: Jest + property-based testing

## 10. Критерии приемки

1. Паттерн генерируется в выбранной тональности (мажор/минор)
2. Длина паттерна настраивается в диапазоне 1-64 шага
3. BPM настраивается в диапазоне 40-240
4. Генерация занимает менее 500ms
5. Экспорт в MIDI работает корректно
6. Пресеты сохраняются и загружаются
7. Валидация отображает ошибки в реальном времени
8. Интерфейс отзывчив при работе с паттернами
